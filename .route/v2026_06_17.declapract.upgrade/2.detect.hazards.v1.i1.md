# 2.detect.hazards — hazard report

## test hazards

### jest 29 → 30 (major version)

- **what changed**: jest upgraded from 29.3.1 to 30.2.0
- **why it may break**: major version bump, potential API changes, assertion behavior changes
- **how to verify**: `rhx git.repo.test --what unit --mode apply`

### ts-jest → @swc/jest

- **what changed**: transform in jest configs changed from ts-jest to @swc/jest
- **why it may break**: different typescript compilation behavior, decorator handling, source maps
- **how to verify**: run tests and check for compilation errors

### core-js removed from setupFiles

- **what changed**: `setupFiles: ['core-js']` removed from jest configs
- **why it may break**: if tests rely on polyfills (Promise.finally, Array.includes, etc.), they may fail
- **how to verify**: run tests in node environment without polyfills

### test-fns reporter added

- **what changed**: added `test-fns/slowtest.reporter.jest` to reporters
- **why it may break**: unlikely, but reporter errors could fail test runs
- **how to verify**: run tests and observe output

### test scripts now use changedSince=main by default

- **what changed**: test:unit and test:integration now skip unchanged files unless `THOROUGH=true`
- **why it may break**: may miss test failures caused by dependency changes in unchanged files
- **how to verify**: run with `THOROUGH=true` to run full suite

## cicd hazards

### aws auth changed to oidc

- **what changed**: aws credentials via secrets → oidc role assumption
- **why it may break**: if github repo not configured for oidc, auth will fail
- **how to verify**: check `.github/workflows/.test.yml` inputs, verify oidc setup in aws

### ubuntu version pinned

- **what changed**: `ubuntu-latest` → `ubuntu-24.04`
- **why it may break**: unlikely, but system packages differ between versions
- **how to verify**: push and observe cicd runs

### actions upgraded (v3 → v4)

- **what changed**: checkout@v3 → v4, cache@v3 → v4, setup-node@v3 → v4
- **why it may break**: unlikely, but input/output names may have changed
- **how to verify**: push and observe cicd runs

### cache key source changed

- **what changed**: cache key now from `needs.install.outputs.node-modules-cache-key` instead of `hashFiles('**/package-lock.json')`
- **why it may break**: if install job output not set correctly, cache miss on every run
- **how to verify**: check `.github/workflows/.install.yml` outputs

## path hazards

### acceptance/ → blackbox/ rename

- **what changed**: `acceptance/environment.ts` moved to `blackbox/environment.ts`
- **why it may break**: imports from old path will fail
- **how to verify**: `grep -r "acceptance/environment" src/` — already verified no imports found

## config hazards

### eslint → biome

- **what changed**: eslint removed, biome added as linter
- **why it may break**: different rule sets, extant code may fail new lint rules
- **how to verify**: `rhx git.repo.test --what lint`

### package manager npm → pnpm

- **what changed**: `package-lock.json` deleted, `pnpm-lock.yaml` added, `packageManager: "pnpm@10.24.0"` set
- **why it may break**: ci scripts that run `npm ci` will fail
- **how to verify**: check `.github/workflows/.install.yml` uses pnpm

### prettier removed from format checks

- **what changed**: `test:format` now runs biome format instead of prettier
- **why it may break**: different format rules, extant snapshots may differ
- **how to verify**: `rhx git.repo.test --what format`

### env var rename in integration tests

- **what changed**: `I_KNOW_WHAT_IM_DOING` → `I_KNOW_THE_RISKS`
- **why it may break**: if scripts rely on old env var name
- **how to verify**: search for old env var usage in scripts

### tsconfig path aliases added

- **what changed**: added `@/*` and `@src/*` path aliases
- **why it may break**: imports using aliases won't work at runtime unless tsc-alias processes them
- **how to verify**: `npm run build` and check dist output

### tsconfig target change

- **what changed**: target es6 → es2020
- **why it may break**: different output syntax; consumers on old node versions may fail
- **how to verify**: check engines.node in package.json aligns

### tsconfig module settings

- **what changed**: added `module: "node16"` and `moduleResolution: "node16"`
- **why it may break**: different import lookup behavior
- **how to verify**: build and check imports work

# 3.2.reflect.test.defects.yield

## classifications

### defect 1: blackbox/environment.ts

**classification**: practice bug

**analysis**: the declapract-typescript-ehmpathy template scaffolds `blackbox/environment.ts` which imports `../src/utils/environment`. this file does not exist in all repos. the template makes an incorrect assumption about repo structure.

**action**: file issue or fix in declapract-typescript-ehmpathy to make the blackbox template more flexible or remove the environment import.

### defect 2: src/index.ts type exports

**classification**: expected behavior (not a defect)

**analysis**: the upgrade changes tsconfig to extend `@tsconfig/strictest` which includes `isolatedModules: true`. this is a stricter, better practice. repos need to update their exports to use explicit `export type` syntax.

this is not a bug — it's an intentional stricter config that surfaces pre-extant issues (type re-exports that should have been explicit).

**action**: none. this is expected behavior from a stricter tsconfig.

### defect 3: src/index.ts biome format

**classification**: expected behavior (not a defect)

**analysis**: the upgrade replaces eslint+prettier with biome. biome has different format rules. run `npm run fix` to auto-format the code.

this is not a bug — it's expected when a new formatter is introduced.

**action**: none. auto-format handles it.

### defect 4: dpdm cycles in aws-sdk v2

**classification**: repo quirk

**analysis**: this repo used aws-sdk v2 (deprecated) which has internal circular dependencies. the dpdm lint check exposed these cycles. the fix was to migrate to AWS SDK v3.

this is specific to this repo. most repos either:
- don't use aws-sdk
- already use v3
- use a different dependency with cycles

**action**: none. this is repo-specific. the exclusion of `@aws-crypto` in `.dpdmrc.yaml` is acceptable for internal AWS SDK transitive dependencies.

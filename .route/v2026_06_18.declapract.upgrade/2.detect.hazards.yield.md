# 2.detect.hazards.yield

## test hazards

### 1. jest 29 → 30 upgrade

**what changed**: jest upgraded from 29.3.1 to 30.2.0

**why it may break**: jest 30 is a major version bump; api changes, snapshot format changes, or matcher behavior changes could cause test failures

**how to verify**: `rhx git.repo.test --what unit --mode apply`

### 2. ts-jest → swc/jest

**what changed**: test transformer changed from ts-jest to @swc/jest in all jest configs

**why it may break**: swc has different transpilation behavior than ts-jest; edge cases in type handle or decorators could fail

**how to verify**: run all test suites

### 3. acceptance → blackbox rename

**what changed**: `acceptance/environment.ts` renamed to `blackbox/environment.ts`

**why it may break**: any imports that reference `acceptance/` will fail

**how to verify**: `rhx git.repo.test --what types`

### 4. source file format changes (unstaged)

**what changed**: biome reformatted source files — import reorder, `import type`, `.flatMap()` instead of `.map().flat()`

**why it may break**: unlikely to break, but `.flatMap()` is ES2019+ — verify node version compatibility

**how to verify**: `rhx git.repo.test --what unit --mode apply`

## cicd hazards

### 5. eslint → biome migration

**what changed**: `.eslintrc.js` deleted, `biome.jsonc` added; all lint scripts now use biome

**why it may break**: lint rules differ between eslint and biome; CI may fail on different lint errors

**how to verify**: `rhx git.repo.test --what lint`

### 6. workflow file changes

**what changed**: multiple github workflow files modified; new workflows added (provision, declastruct)

**why it may break**: workflow syntax or step changes could fail CI

**how to verify**: review diff of `.github/workflows/`; push and watch CI

### 7. test commands changed

**what changed**: test scripts now include conditional flags `--changedSince=main` when not in thorough mode

**why it may break**: on feature branches, only changed files tested; could miss regressions in unchanged files

**how to verify**: CI runs with THOROUGH=true for full coverage

## path hazards

### 8. eslintrc.js deleted

**what changed**: `.eslintrc.js` removed

**why it may break**: any scripts or configs that expect eslint will fail

**how to verify**: grep for `eslint` references

### 9. prettier.config.js deleted

**what changed**: `.prettierignore` and prettier config removed

**why it may break**: any scripts that reference prettier directly will need update

**how to verify**: grep for `prettier` references

## config hazards

### 10. tsconfig changes

**what changed**: tsconfig.json and tsconfig.build.json modified

**why it may break**: stricter compiler options or path changes could cause type errors

**how to verify**: `rhx git.repo.test --what types`

### 11. new prod dependencies

**what changed**: added `domain-objects`, `helpful-errors`, `type-fns` as prod dependencies

**why it may break**: bundle size increased; unlikely to break functionality

**how to verify**: check package exports still work
